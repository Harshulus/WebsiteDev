How the Web Works


Clients and Servers

The web consists of a client and a server which interact with each other. The interaction between the server and client is just a simple request and respond system, where the client requests something, e.g. clicking on a hyperlink. The server responds by redirecting the user. The clients are typically internet users who access the website through a browser. Servers are computers that store webpages, sites, or apps.



Other Parts of the Web

  • Internet connection - Allows you to send and receive data on the web.
  • Transmission Control Protocol and Internet Protocol (TCP/IP): It is the communication protocols that define how data should travel across the internet.
  • Domain Name Server (DNS) - DNS is a database which keeps track of the IP addresses of websites. It is easier to remember the addresses, e.g. www.google.com.au, rather than the IP address of the website.
  • Hypertext Transfer Protocol (HTTP) - It is an application protocol that defines a language for clients and servers to speak to each other.
  • Uniform Resource Locators (URL): It specifies the protocol, hostname and filename.
  • Code files - The language used for the files is mainly HTML, CSS and JavaScript
  • Assets - This is all the multimedia files such as images, audio, video, etc.
  • Packets - It is the terminology used to describe the data sent to the client from the server.

The Interaction Between Server and Clients

  1. The browser will find the address of the server from the DNS server.
  2. The browser will request for a copy of the website using a TCP/IP.
  3. The server will approve of the request and send a message consisting of “200 OK”.
  4. The server will then send the files of the website to the browser in a series of small chunks called data packets.
  5. The browser compiles the chunks into the whole website and displays for the user.
